# User Guide

このガイドは、家庭内でアプリを使う人向けの説明です。

## 起動

管理者がサーバー端末でDocker Composeを起動します。

```bash
docker compose up
```

同じ端末で使う場合は次のURLを開きます。

```text
http://localhost:3000
```

家庭内LANの別端末やスマホから使う場合は、サーバー端末のローカルIPアドレスを使います。

```text
http://<local-ip>:3000
```

例:

```text
http://192.168.1.20:3000
```

## 初期ログイン

初期状態ではseedでownerユーザーが作成されます。

```text
email: owner@example.local
password: change-me
```

管理者は実運用前に `.env` の `AUTH_SECRET` と `INITIAL_OWNER_PASSWORD` を変更してください。既にDBが作られている場合、seedは既存ownerのパスワードを上書きしません。初期値を変えてやり直す場合は、DBをバックアップまたは削除してから再起動します。

## 画面

### 今日買う

現在の買い物リストです。追加内容とチェック操作はDBに保存され、再読込や再起動後も残ります。

表示されるもの:

- 商品名
- 数量
- 店舗グループ
- 優先度
- メモ
- 完了済みの折りたたみ
- 「買うものを追加」からの商品名、数量、店舗、優先度、メモの登録

スーパー店内で片手でも操作しやすいよう、チェックボタンは大きめにしています。

### いつもの

定番品や消耗品の一覧です。「定番品を登録」から商品を追加でき、各商品の＋ボタンで今日の買い物へ追加できます。同じ商品が未購入のまま存在するときは重複追加しません。

### 残量

各商品の残量状態を確認・更新する画面です。「十分」「少ない」「空」「未確認」を選んで保存できます。上部の補充候補件数は「少ない」「空」の商品数です。

### 設定

現在のログインアカウント、家庭名、ロールを確認できます。ログアウトもここから行います。

## ログアウト

設定画面の「ログアウト」を押します。ログアウト後はログイン画面に戻ります。

## データ保存について

ユーザー、家庭、商品、残量、店舗、買い物リストがSQLite DBに保存されます。

現時点で保存されるもの:

- 初期ownerユーザー
- 家庭情報
- 家庭メンバー情報
- 定番品と標準店舗
- 在庫レベル
- 買い物リストの中身とチェック済み状態

まだ保存されないもの:

- 購入履歴
- 通知設定

自宅サーバーでのバックアップと復元は[Self-hosting Guide](self-hosting.md)を参照してください。

## 運用上の注意

- アプリは家庭内LANで使う前提です。インターネットへ公開しないでください。
- `.env` やDBファイルをGitHubにコミットしないでください。
- 初期パスワード `change-me` のまま実運用しないでください。
- スマホで使う場合は、同じWi-Fiに接続してからサーバー端末のIPアドレスで開いてください。
